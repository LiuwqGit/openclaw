## What Problem This Solves

Fixes issue #103956: the `session.pruneAfter` configuration setting is silently ignored during normal gateway operation. Sessions older than the configured prune cutoff grow unbounded regardless of the `pruneAfter` duration.

## Why This Change Was Made

During load-time session store maintenance, stale-entry pruning (`pruneStaleEntries`) was gated by `Object.keys(store).length > maintenance.maxEntries`. This meant age-based pruning only fired when the store exceeded its entry count limit. If a user had fewer sessions than `maxEntries` (default 500), no pruning ever happened — `pruneAfter` effectively became dead configuration.

The fix separates the pruning trigger from the entry-count gate, matching the established pattern in `applyEnforcedMaintenance()` in `store-maintenance-operations.ts` which already runs `pruneStaleEntries` unconditionally in enforce mode.

## User Impact

Users who configure `session.maintenance.pruneAfter` will now see sessions older than the configured duration automatically removed during load-time maintenance, regardless of how many sessions exist. The entry-count cap (`maxEntries`) continues to work exactly as before — it is still gated by `shouldRunSessionEntryMaintenance`.

## Evidence

**Root cause**: In `src/config/sessions/store-load.ts`, the `pruneStaleEntries` call at load-time was inside an `if` condition `maintenance.mode === "enforce" && Object.keys(store).length > maintenance.maxEntries`. Removing the entry-count gate restores independent age-based pruning.

The fix is a structural mirror of `applyEnforcedMaintenance()` in `store-maintenance-operations.ts`, which already runs `pruneStaleEntries` unconditionally (when mode is `"enforce"`):

```typescript
// store-maintenance-operations.ts (correct behavior — runs pruning unconditionally)
const pruned = pruneStaleEntries(store, maintenance.pruneAfterMs, { ... });
const countAfterPrune = Object.keys(store).length;
const shouldRunCapMaintenance = shouldRunSessionEntryMaintenance({ ... });
```

```typescript
// store-load.ts (after fix — same pattern)
pruned = pruneStaleEntries(store, maintenance.pruneAfterMs, { ... });
const countAfterPrune = Object.keys(store).length;
capped = shouldRunSessionEntryMaintenance({ ... }) ? capEntryCount(...) : 0;
```
